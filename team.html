Functional Organizational Structure

To allow group members to focus on the areas of the project they are most interested in and more professional at, we decided to use a functional organizational structure. We create sub-teams which will work on specific tasks where each sub-team has a team leader who supervises the team to achieve goals in the project plan.

Sub-team Responsibilities Team Leader Team Members
Project Management Responsible for the effective management of the groups’ resources and keeping track of deadlines. Will also be responsible for drafting reports and collecting feedback from clients. Daniel Stirling, Gagan

Productivity Tools

To aid our group collaboration we use a range of productivity tools:

  1. MS Teams - main method of communication
    • We make the most individual channels on Teams to effectively communicate information in specific areas. Such as the General channel being reserved for announcements and channels for each sub-team.
    • We have a Weekly meeting with all team members on Teams in order to see at what stage we are at and discuss the work planned for each member for the week ahead
  2. Trello - task allocation & progress tracking
    • This platform provides a flexible and transparent board which will allow us to track what individual members are working on.
    • It also allows us to track deadlines and provide an overview of upcoming events.
    • It is nicely integrated into Teams which our communication and task allocation are all within one application.
  3. Github - code sharing and version control
    • We will use repositories to split code up into individual areas (such as the Webots simulation, Hardware ROS code and the Website code).
    • Sub-teams will be responsible for their respective repositories but should strive to only submit high-quality code.
    • To ensure code quality each Github commit should be peer-reviewed by another member of the group. If the code does not meet the sub-team’s coding standards then the code should be sent back to the author with constructive comments on how to rectify their code
  4. Overleaf - main LaTeX editor
    • It allows us to work as a team simultaneously on project reports.
    • All team members can view, recommend and implement changes in their area of expertise at the same time
